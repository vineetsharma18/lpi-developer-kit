## What I Built
I built a Decision Tradeoff Agent that helps users compare different options and make better decisions based on their priorities.

Instead of just returning information, the agent asks what matters most to the user (such as salary, interest, or stability) and then performs a structured comparison between the options.

It combines knowledge from the LPI sandbox and generates a tradeoff analysis that is more aligned with real-world decision-making rather than generic advice.

## LPI Tools Used

The agent uses the following LPI tools:

- query_knowledge  
- get_insights  

These tools provide:

• background knowledge about the decision context  
• scenario-based insights for practical guidance  

The agent combines both to generate a structured tradeoff analysis.

---

## Explainability

The agent explicitly shows how the output is generated by:

• presenting knowledge summaries  
• displaying insights retrieved from LPI tools  
• structuring the comparison between options  
• listing the sources used  

Workflow:

User Input  
→ Agent  
→ query_knowledge + get_insights  
→ Tradeoff Analysis  
→ Final Recommendation  

---

## Error Handling

The agent handles basic input validation.

If the user enters an unclear or incomplete decision, the agent still provides a fallback response and avoids breaking execution.

---

## Design Choices

Instead of building a generic advisor, I focused on decision-making.

I added:

• option parsing (e.g., "AI vs Web Dev")  
• priority-based reasoning  
• structured comparison  

This makes the agent more useful and closer to how real decisions are made.
